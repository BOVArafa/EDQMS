# Development and Implementation

!!! Warning "Digital Enrollment"

    Successful delivery of this phase depends on aligned planning and execution with the Digital organization.

Phase 3 is the deployment phase of the EDQMS project. It begins after the completion of Phase 2's Prototype Implementation Assessment (Deliverable 003) and the Target-State Solution Architecture (Deliverable 005), which together define exactly what needs to be built and how.

## Purpose

The purpose of Phase 3 is to transform the validated SOP prototype into a production-ready system that the Global Engineering Hub can operate as part of its daily quality management practice.

Where Phase 2 validates whether the methodology works, Phase 3 makes it permanent. The engineering team moves from interacting with prototype lists to working within a system that is integrated, access-controlled, partially automated, and maintained as a live operational tool.

## What Phase 3 Addresses

Phase 2's scope intentionally excludes several capabilities that are required for production use but would introduce unnecessary complexity during prototype validation. These items are documented in the Target-State Solution Architecture and form the primary development backlog for Phase 3:

| Capability | Rationale |
| :--- | :--- |
| User authentication and role-based access control | Ensures that each team member interacts with the system according to their defined role and authority level |
| Automated workflow triggers | Replaces manual activation with event-driven automation (via Power Automate or equivalent), reducing response time and human error |
| Integration with Siemens Energy's SAP or internal ERP systems | Connects the quality management layer to transactional data already flowing through existing systems |
| Nonconformity tracking (ISO 9001:2015 §10.2) | Completes the improvement cycle: detected deviations are logged, classified, assigned, and closed within the same system |
| Documented Information lifecycle management (ISO 9001:2015 §7.5) | Distinguishes maintained records (living process documents) from retained records (historical evidence of conformity), enabling full audit readiness |

## Proposal Status

A formal proposal for Phase 3 does not yet exist. It will be developed after the completion of Phase 2, incorporating the findings from the Prototype Implementation Assessment and the Target-State Solution Architecture. The scope, timeline, and investment for Phase 3 will be defined at that point based on the validated requirements.

This is by design: scoping Phase 3 before Phase 2 is complete would introduce risk. The implementation architecture must be grounded in what the prototype validation actually reveals, not in assumptions made before the validation work is done.

## Expected Outcomes

When Phase 3 is complete, the Global Engineering Hub will have:

- A fully operational, ISO 9001:2015 aligned quality management system embedded in its daily engineering workflows
- A system capable of being replicated across other business units or regions as the hub expands
- Complete traceability from every operational event to the requirement, process, and procedure that governs the response
- A governance structure in which new team members can onboard into a documented system rather than relying on knowledge transfer from experienced colleagues

Phase 3 is where the answer to the project question becomes a permanent capability — not a prototype, but a system.
