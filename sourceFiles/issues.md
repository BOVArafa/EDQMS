Durante o etapa de validacao (conforme item 003 da proposta PRP-C-0017), deve ser criado um documento de nao-conformidade para os procedimentos que apresentarem problemas de execucao.

Exemplo de problemas de execucao:
- Em um step relacionado a cadastro de documento, alguns usuarios tem acesso a ferramenta de cadastro e outros nao.
- Uma determinada atividade deve ser cadastrada mas o ambiente para cadastro ainda nao foi configurado.

Essa tabea de nao-conformidade sera criada no microsoft list e contara com os seguintes parametros

- Procedure: qual o procedimento afetado
- Issues: a descricao dos problemas encontrados no procedimento
- Impact: qual a severidade do problema (critical, major ou minor)

A decisao de se criar mais de uma nao-conformidade por procedimento deve estar relacionada ao nivel de impacto dos problemas. OU seja, nao se pode misturar problemas que possuam impacto distintos para um mesmo processo.

Este cadastro de nao conformidade sera realizado nesta [lista de sharepoint](https://neundesign-my.sharepoint.com/:l:/g/personal/bova_neun-design_com_br/JAAmTRls-hSWRqNzDRQ3efD8AXa9hNlfsdBISi06YuHAShg?e=XiUQRD)
